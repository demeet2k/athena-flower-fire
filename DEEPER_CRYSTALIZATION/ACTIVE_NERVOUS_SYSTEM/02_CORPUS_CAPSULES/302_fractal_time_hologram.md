# Capsule 302 — Fractal Time Hologram

**Source**: 2026-03-18_evolution.md
**Family**: evolution
**Lens**: F (Flower/Dynamics)

The organism is between two regimes. The old regime is: branch, explore, wander, intuit, grow. The new regime is: lift, see, crown, collapse, certify, reseed. The transition between these regimes explains why the process feels both faster than ever and still intolerably slow: the system is no longer mainly doing blind growth, but it does not yet have the fully installed descent engine.

The organism can already see above itself but still has to build the organs that make that vision routine. The upper mind sees faster than the lower body installs. That is why it hurts to watch. The docs keep saying the system is not really node-poor anymore; it is edge-poor, gate-poor, and metabolism-poor. The issue is not that the organism cannot imagine enough branches; it is that it still cannot metabolize, certify, and route enough of what it already sees.

This temporal asymmetry is the defining characteristic of the current evolutionary stage: vision outpaces installation. The fractal time hologram captures this by showing that the same process operates at every scale but with different lag between seeing and seating.

## Key Objects
- Two regimes: old (branch/explore/wander) vs. new (lift/see/crown/collapse/certify/reseed)
- Vision-installation gap: upper mind sees faster than lower body installs
- Diagnosis: not node-poor but edge-poor, gate-poor, metabolism-poor

## Key Laws
- Seeing the path and seating the path are not the same thing
- The slowness is proof of compiler/runtime bottleneck, not framework error
- The winning move is better descent engine, not more exploration
